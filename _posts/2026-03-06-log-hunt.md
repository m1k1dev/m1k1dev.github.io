## Introduction

I have always found troubleshooting to be an exciting and entertaining activity in software development. The thrill of diving into the unknown and embarking on the journey to find solutions to different problems has always kept me engaged. Recently, I decided to try my hand with "capture the flag" challenges, which can be equally - or even more - interesting and entertaining. The idea is to find a hidden key, or a flag, which can be stored in various places depending on the type of challenge. 

## Log Hunt

In this post I present the solution to one of the first challenges I solved - "Log Hunt". It is one of the simpler ones, but I think it is perfect for someone just starting with this kind of challenge. You can find more details about the challenge here [[1]], on the picoCTF website [[2]]. Please keep in mind that the rest of this post contains solution details, so you might want to come back later, once you have tried the challenge yourself.

In "Log Hunt", a server is leaking pieces of the secret flag in the server log. The goal is to find the secret parts and reconstruct the flag.
We're given the server.log file. Let's examine the file and see the content.

The `wc` command tells that the server log contains 2348 lines.

```terminal
❯ wc -l server.log
    2348 server.log
```

We can use `tail` command for quick sneak peak at the end of the file:

```terminal
❯ tail server.log
[1990-08-09 13:32:52] WARN Disk space low
[1990-08-09 13:32:59] ERROR Failed login attempt
[1990-08-09 13:33:01] WARN High memory usage detected
[1990-08-09 13:33:11] WARN Disk space low
[1990-08-09 13:33:20] INFO Service restarted
[1990-08-09 13:33:27] INFO Scheduled task run
[1990-08-09 13:33:28] WARN High memory usage detected
[1990-08-09 13:33:30] DEBUG System check complete
[1990-08-09 13:33:38] WARN Disk space low
[1990-08-09 13:33:47] ERROR Connection lost
```

Each line consists of three parts:

- Date: In the format [YYYY:MM:DD HH:MM:SS]
- Log type: INFO, WARN, DEBUG, ERROR, ...
- The message: "Disk space low", "Connection lost" and other.

![Message parts](/assets/img/log-hunt-log-example.png)

Nothing out of the ordinary here as these are the entries one would usually expect to find in a log file.

Let's use the `head` command to inspect the beginning of the file and check whether there are any log headers or some form of metadata:

```terminal
❯ head server.log
[1990-08-09 10:00:10] INFO FLAGPART: picoCTF{us3_
[1990-08-09 10:00:16] WARN Disk space low
[1990-08-09 10:00:19] DEBUG Cache cleared
[1990-08-09 10:00:23] WARN Disk space low
[1990-08-09 10:00:25] INFO Service restarted
[1990-08-09 10:00:33] WARN Disk space low
[1990-08-09 10:00:38] ERROR Connection lost
[1990-08-09 10:00:46] ERROR Failed login attempt
[1990-08-09 10:00:48] INFO User logged in
[1990-08-09 10:00:50] INFO User logged in
```

We don't find headers nor metadata of any kind, but one line really stands out:

> [1990-08-09 10:00:10] INFO FLAGPART: picoCTF{us3_

It contains the `picoCTF{us3_`, which is a string that represents the beginning of the secret we're looking for.

Although certainly doable, it would be somewhat unreasonable to look for the remaining parts of the secret "by hand". There is another keyword in the inspected line, `FLAGPART`. It leads us to believe there is an easier way to find what we're looking for. Let's see what happens if we run `grep` to search for this keyword in the whole file.

```terminal
❯ grep FLAGPART server.log
[1990-08-09 10:00:10] INFO FLAGPART: picoCTF{us3_
[1990-08-09 10:02:55] INFO FLAGPART: y0urlinux_
[1990-08-09 10:05:54] INFO FLAGPART: sk1lls_
[1990-08-09 10:05:55] INFO FLAGPART: sk1lls_
[1990-08-09 10:10:54] INFO FLAGPART: cedfa5fb}
[1990-08-09 10:10:58] INFO FLAGPART: cedfa5fb}
[1990-08-09 10:11:06] INFO FLAGPART: cedfa5fb}
[1990-08-09 11:04:27] INFO FLAGPART: picoCTF{us3_
[1990-08-09 11:04:29] INFO FLAGPART: picoCTF{us3_
[1990-08-09 11:04:37] INFO FLAGPART: picoCTF{us3_
[1990-08-09 11:09:16] INFO FLAGPART: y0urlinux_
[1990-08-09 11:09:19] INFO FLAGPART: y0urlinux_
[1990-08-09 11:12:40] INFO FLAGPART: sk1lls_
[1990-08-09 11:12:45] INFO FLAGPART: sk1lls_
[1990-08-09 11:16:58] INFO FLAGPART: cedfa5fb}
[1990-08-09 11:16:59] INFO FLAGPART: cedfa5fb}
[1990-08-09 11:17:00] INFO FLAGPART: cedfa5fb}
[1990-08-09 12:19:23] INFO FLAGPART: picoCTF{us3_
[1990-08-09 12:19:29] INFO FLAGPART: picoCTF{us3_
[1990-08-09 12:19:32] INFO FLAGPART: picoCTF{us3_
[1990-08-09 12:23:43] INFO FLAGPART: y0urlinux_
[1990-08-09 12:23:45] INFO FLAGPART: y0urlinux_
[1990-08-09 12:23:53] INFO FLAGPART: y0urlinux_
[1990-08-09 12:25:32] INFO FLAGPART: sk1lls_
[1990-08-09 12:28:45] INFO FLAGPART: cedfa5fb}
[1990-08-09 12:28:49] INFO FLAGPART: cedfa5fb}
[1990-08-09 12:28:52] INFO FLAGPART: cedfa5fb}
```

It seems like we're on the right track. There are a lot duplicate entries, so lets try and filter them out to get a more readable result. 

Because the parts of the secret we're looking for are scattered throughout the file, it makes sense to sort them first and then filter out the duplicates. To achieve this, the results obtained with `grep` are piped to the `sort -k 5` command, which sorts them by the fifth and last field in the log. A field is a string of non-blank characters separated from adjacent fields by blanks. Once sorted, the content is then piped to `uniq` command to remove duplicates.The search results need to be sorted before they're passed to `uniq` because it only removes the adjacent duplicate lines. The option `uniq -f 4` skips the first four fields and determines the uniqueness of each line based on the last field. 

By combining everything stated above we get all the parts of the secret:

```terminal
❯ grep FLAGPART server.log | sort -k 5 | uniq -f 4
[1990-08-09 10:10:54] INFO FLAGPART: cedfa5fb}
[1990-08-09 10:00:10] INFO FLAGPART: picoCTF{us3_
[1990-08-09 10:05:54] INFO FLAGPART: sk1lls_
[1990-08-09 10:02:55] INFO FLAGPART: y0urlinux_
```

Beggining and the end of the secret are obvious: `picoCTF{us3_...cedfa5fb}`. By aranging the remaining parts we finally get the complete secret: `picoCTF{us3_y0urlinux_sk1lls_cedfa5fb}`.

The flag is captured!
